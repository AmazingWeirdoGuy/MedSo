That error means your CMS is trying to load **HTML instead of YAML** for `config.yml` (it sees `<!DOCTYPE html>` at the top). This happens when the file path is wrong or the file isn’t being served from the place the CMS expects.

Do these fixes:

### 1) Put the files in the exact public path

Vite copies anything in `client/public` straight to the site root at build time. Decap/Netlify CMS looks for `/admin/config.yml` by default.

**Make this structure:**

```
client/
  public/
    admin/
      index.html
      config.yml
```

### 2) Minimal `admin/index.html`

```html
<!doctype html>
<html>
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width,initial-scale=1" />
    <title>Admin</title>
  </head>
  <body>
    <!-- Loads Decap CMS and it will auto-fetch /admin/config.yml -->
    <script src="https://unpkg.com/decap-cms@^3.0.0/dist/decap-cms.js"></script>
  </body>
</html>
```

### 5) Check your Vercel rewrites

If you added a `vercel.json` to rewrite everything to `/`, make sure **/admin** is not being rewritten to your SPA root. You can add these so `/admin` serves the static CMS page:

```json
{
  "rewrites": [
    { "source": "/api/(.*)", "destination": "/api/$1" }
  ],
  "routes": [
    { "src": "/admin", "dest": "/admin/index.html" },
    { "src": "/admin/(.*)", "dest": "/admin/index.html" }
  ]
}
```

(If you don’t have `vercel.json`, Vercel will serve `/admin/index.html` automatically as long as that exact path exists in your output.)

* No tabs in YAML; use spaces.
* No BOM/hidden characters at the top of the file.
* The first line of the file must be `backend:` (not `<html>`).
